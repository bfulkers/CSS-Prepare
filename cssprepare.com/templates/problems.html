Having problems?

CSS Prepare should know about all of the properties defined in the CSS 2.1 specification. There are bound to be edge cases, however. If you find something you believe to be valid CSS but this complains, please report the problem (as described below).

Known problems

There are a few of the more esoteric parts of the CSS grammar that will trip CSS Prepare up. For example, being able to use newlines inside of selectors (see the last example in section 4.1.7 of the specification). Whilst this may make CSS Prepare a non-conforming user agent because that means it fails on point 3. For me it is not as a high priority as supporting CSS3 properties that actually work in modern browsers.I have never seen anyone use it in the wild. But please, prove me wrong and I'll fix it.

CSS Hacks

There are a couple of common css hacks (used to target earlier versions of Internet Explorer) that should work fine, including the star and underscore hacks. The star HTML hack works fine, as it is valid CSS.

Another common hack is to use the Microsoft-proprietary zoom:1 declaration to trigger hasLayout. This will work fine if used with either of the star or underscore hacks, but used alone you will get an invalid property error. The same goes for any filter or other IE-specific declarations (I consider this to be the correct behaviour, as in IE8 Microsoft have corrected these to be -ms vendor extensions).

Unknown problems

If you have any problems with this, you can contact me in one of three ways:

  1. First, just use the Mark result as containing a problem button on the output page to flag the output as something I need to examine if you think it is fairly obvious what might be wrong.
  2. If you think the problem needs a little more explanation, feel free to send me an email describing your problem at norm@cackhanded.net. Please include the permanent URL of your problem output in the email.
  3. Or if you prefer, use github to report an issue with the code. Please include the permanent URL of your problem output. (And if you feel up to it, by all means fork it, fix the code and send me a pull request.)
